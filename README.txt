РЕШЕНИЕ ПРЯМОЙ И ОБРАТНОЙ ЗАДАЧИ ТЕПЛОПРОВОДНОСТИ (Тестовое задание)

Реализует решение прямой задачи теплопроводности с источником схемой с расщеплением. Граничные условия неоднородные, первого рода.
Автор: Шумарин Валерий

[УСТАНОВКА]
  1) Сойдайте в корне проекта папку build
  2) Соберите проект в этой папке
[ИСПОЛЬЗОВАНИЕ]
  1) В директории ./conf уже есть два конфигурационных файла для обеих типов задач. Настроить их под свою задачу
  2) Из директории ./build запустить проект одним из двух способов:
    ./thermal ../conf/test.conf     (для прямой задачи)
    ./thermal ../conf/test_i.conf   (для обратной задачи)
[КОНФИГУРАЦИЯ]
  ExactSolution             - класс чтения решения из txt
  ConfigReader              - класс чтения конфигурационного файла
  Grid                      - класс сетки
  BoundaryConditions        - класс граничных условий
  SourceFunction            - класс функции источника
  ThermalConductivity       - класс коэффициентов
  DirectSolver              - класс решения прямой задачи
  Solver                    - класс решения прямой задачи на одном временном шаге
  InverseSolver             - класс решения обратной задачи
  SingleStepInverseSolver   - класс решения обратной задачи на одной итерации
  VTKSaver                  - класс сохранения решения в vtk
  TempSaver                 - класс сохранения решения в txt
  ParserConst               - вспомогательный класс парсера со значениями констант
[СТРУКТУРА]
  ./conf                    - хранилище конфигурационных файлов 
  ./include                 - заголовочные файлы
  ./src                     - исполняемые файлы, включая main.cpp
  ./parsers                 - парсеры, подключен только один muParser для считывания формул из строк
  ./output                  - хранилище результатов
    ./vtk               - в vtk
    ./txt               - в txt
  CMakeLists.txt            - настройки сборки
[РЕКОМЕНДАЦИИ]
  1) Для прямой задачи не рекомендую ставить значения коэффициентов, больших 0,01 - 0,02, так как распространение тепла происходит слишком быстро
     и схема не успевает за изменением функции. Если необходимо поставить больше - то либо увеличивайте размеры области, либо уменьшайте dt
  2) Для обратной задачи - настройка коэффициентов очень тонкая, подобрать сложно. Если будете подбирать, то старайтесь держать скорость обучения
     на уровне рекомендованной. Ее увеличение ведет к расхождению - выходу в отрицательные значения, а уменьшение - к сходимости гораздо раньше.
